# 구현 요청 프롬프트

## 목적

AI 에이전트에게 현재 프로젝트 문맥을 정확히 전달하고, 작은 단위로 구현을 진행시키기 위한 프롬프트입니다.

## 프롬프트

```text
우리는 React 핵심 기능을 직접 구현하는 프로젝트를 진행 중이고, 현재 기준 브랜치는 develope입니다.

먼저 아래 문서를 반드시 읽고 현재 설계 의도를 반영해 주세요.
- docs/README.md
- docs/requirements.md
- 필요 시 docs/01-virtual-dom.md ~ docs/06-render-commit-phase.md

프로젝트 목표는 다음과 같습니다.
- React의 핵심 기능인 Component, State, Hooks를 직접 구현한다.
- Virtual DOM + Diff + Patch도 직접 구현한다.
- 최종적으로는 비트코인 실시간 데이터를 1초 단위로 갱신해서 보여주는 화면을 만든다.
- 참고 UI 예시는 docs/assets/binance-btc-realtime-example.png 및 docs/requirements.md의 설명을 따른다.

중요한 제약 조건:
- 반드시 함수형 컴포넌트 기반으로 구현
- 상태는 최상위 루트 컴포넌트에서 관리
- 자식 컴포넌트는 props만 받아 렌더링하는 stateless 순수 함수 형태
- FunctionComponent 클래스 중심 구조 유지
- hooks 배열과 hook index 기반으로 상태를 관리
- 상태 변경 시 리렌더링이 자동으로 트리거되어야 함
- Diff/Patch를 통해 필요한 부분만 DOM 업데이트
- 기존 `react`, `react-dom` 패키지나 React 런타임을 사용하면 안 됨
- 반드시 우리가 직접 개발한 모듈(패키지/라이브러리)만 사용해야 함
- 문서의 설계 방향과 다르게 일반적인 React 구현을 새로 가져오면 안 됨

작업 방식:
1. 먼저 현재 코드베이스를 읽고 지금 어디까지 구현됐는지 파악
2. 내가 지금 구현해야 할 최소 단위를 명확히 정의
3. 해당 단위만 실제 코드로 구현
4. 가능한 테스트나 실행 검증 수행
5. 문서와 코드가 어긋나면 관련 문서도 함께 수정
6. 최종적으로 무엇을 변경했는지와 남은 작업을 간단히 정리

이번 요청의 직접 목표:
- 현재 코드 상태를 기준으로, “비트코인 실시간 데이터를 1초 단위로 갱신해 보여주는 화면”에 도달하기 위해 다음으로 구현해야 할 가장 적절한 단계를 선택해서 진행해 주세요.
- 한 번에 전체를 다 만들려고 하지 말고, 현재 구조에 맞는 가장 작은 단위의 유효한 구현부터 해 주세요.
- 구현 전에 관련 파일과 구조를 먼저 조사한 뒤 바로 코드 변경까지 진행해 주세요.

응답 형식:
- 먼저 현재 상태를 아주 짧게 요약
- 그 다음 바로 구현
- 마지막에 변경 파일, 검증 결과, 다음 단계만 간단히 정리
```
